This webapp harnesses the power of real-world messages, sourced from Figure Eight (now Appen), to bring a technological edge to disaster response efforts. By analyzing messages shared via social media or sent directly to disaster relief organizations, we explore the potential for machine learning to help emergency personnel and responders.

How Does The Model Work

The message classifier is a Linear Support Vector (LinearSVC) model, tailored for Natural Language Processing (NLP). This model is adept at handling high-dimensional data and is particularly well-suited for text classification tasks, such as categorizing disaster response messages.

Preprocessing:

Utilizing NLP techniques like tokenization, lemmatization, and the removal of stopwords, we prepare the messages for analysis. This preprocessing stage is crucial for transforming raw text into a structured format that the model can understand.

Feature Extraction:

The processed text data is then converted into numerical features using techniques like TF-IDF (Term Frequency-Inverse Document Frequency), which helps in highlighting the most relevant words for classification tasks.

Multi-Label Classification:

Our LinearSVC model is trained to perform Multi-Label Classification utilizing binary relevance, so that it can assign multiple categories to a single message. This is essential for disaster response messages, which often contain information pertinent to various emergency services and needs.

Model Training and Evaluation:

We employ techniques like GridSearch and MultilabelStratifiedKFold for hyperparameter tuning and stratified data seperation in order to squeeze the best performace out of the model. The current weighted-f1 score of .35, which leaves much room for improvement. We'll discuss this challenge next.

Future Considerations

There are a number of ways this app could be further improved:

Model Performance:

We can continue to refine our model by exploring more sophisticated NLP methodologies, such as deep learning models and transformers (e.g., BERT, GPT), engineering new features, ensembling methods, and different NLP techniques like word embeddings and semantic analysis.

Multilingual Support:

We could add multilingual support to our webapp, allowing users to input and classify messages in various languages, thereby broadening reach and impact.

Deal With Label Imbalance:

We could implement strategies such as oversampling minority classes, undersampling majority classes, or synthesizing new examples through techniques like SMOTE (Synthetic Minority Over-sampling Technique). Additionally, we could try and incorporate other datasets that represent a broader and more balanced spectrum of disaster scenarios and response categories.

Real-Time Data Streams:

We could explore the possibility of integrating real-time data streams, enabling the webapp to process live data from social media and other communication channels during disasters for immediate analysis and response.
